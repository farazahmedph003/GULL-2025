## Cyberpunk Dark-Mode Redesign Plan (v1.4)

### 1. Lock the app to a single dark theme

- **Unify theme behavior**: Update [`src/contexts/ThemeContext.tsx`](src/contexts/ThemeContext.tsx) so the app always uses `dark` (or a renamed `neo` theme) and ignores system light/dark and manual toggling.
- **Remove light/dark toggle UI**: Find any theme toggle controls (e.g. in settings, `ThemeTestComponent`) and either hide/remove them or repurpose for something else.
- **Clean up body/html classes**: Ensure only one theme class (e.g. `dark` or `neo`) is ever applied on `html` and `body`.

### 2. Redesign the core color system to a cyberpunk dark palette

- **Define new base palette in CSS**: In [`src/index.css`](src/index.css), replace the current light/dark variables with a single set of variables tuned for a dark, neon/cyberpunk look:
- Backgrounds: deep navy / charcoal gradients (e.g. `#050816` → `#020617`).
- Primary accent: vivid cyan/teal (for main actions and highlights).
- Secondary accent: magenta/purple for secondary CTAs and focus states.
- Supporting accent: subtle amber/orange for warnings and emphasis.
- **Create gradient tokens**: Add variables like `--color-surface-gradient`, `--color-sidebar-gradient`, `--color-button-primary-gradient` and use them for cards, sidebars, and primary buttons.
- **Unify text tokens**: Collapse separate light/dark text colors into a single dark-mode-optimized scale (primary, secondary, muted, disabled) with high contrast on the new backgrounds.

### 3. Apply the new palette to global surfaces and layout

- **App shell**: Update `body`, app root, and layout containers (e.g. [`src/App.css`](src/App.css), `Layout`/`AdminLayout`/`Sidebar` components) to:
- Use the new background gradients.
- Use consistent card surfaces (e.g. slightly raised glassy cards with border + soft glow on hover).
- Ensure scrollbars, dividers, and borders use the new dark-border tokens.
- **Typography**: Keep `Outfit`/`Inter` but:
- Set a consistent font stack and weights in `index.css`.
- Adjust heading sizes and letter-spacing to match the bold, futuristic feel.
- **Spacing and radius**: Standardize border-radius and shadow tokens from `index.css` (e.g. `--radius-card`, `--shadow-soft`, `--shadow-strong`) and apply them across cards, modals, and sidebars.

### 4. Update key components to use tokens instead of hard-coded colors

- **Audit components for hard-coded colors**: Search in `src/components` and `src/pages` for any Tailwind color classes (`bg-gray-800`, `text-gray-900`, `bg-white`, etc.) and inline hex colors.
- **Refactor to token-based classes**:
- Introduce utility classes in `index.css` (e.g. `.bg-surface`, `.bg-elevated`, `.text-primary`, `.text-muted`, `.border-subtle`, `.accent-primary`, `.accent-secondary`).
- Replace direct light/dark Tailwind classes in shared components like `AdminLayout`, `AdminSidebar`, `Sidebar`, `TabNavigation`, `GlassmorphicCard`, `GlassStatCard`, `SummaryCard`, `Modal` components with these utilities.
- **Buttons and interactive elements**:
- Standardize primary, secondary, ghost, and destructive button styles (colors, hover, active, focus) using the new palette.
- Ensure all states are high-contrast and readable.

### 5. Remove/neutralize legacy light-mode-specific styling

- **Body light-mode background and patterns**: In [`src/index.css`](src/index.css), remove or comment out `body.light` and `body.light::before` rules, and any `.light`-only tweaks.
- **`dark:` Tailwind modifiers**: Where components use `dark:` variants, simplify to a single style that looks good on the new dark theme and remove the unused light-mode variants.
- **Settings/UI references**: In pages like settings or theme test components, remove any copy or controls that refer to “Light mode”, “Dark mode”, or system-following mode.

### 6. Verify across key screens and polish

- **Visual QA on main flows**: Manually verify visual consistency on:
- `AdminAkraPage`, admin dashboard, and other high-traffic admin pages.
- User dashboard, sidebar navigation, modals, and tables/grids.
- **Accessibility & contrast pass**:
- Check primary text, error/warning states, and buttons for sufficient contrast against the new backgrounds.
- Adjust individual token values if any area feels low-contrast or harsh.
- **Micro-polish**:
- Add subtle glows or accent borders on hover for important components (tabs, active sidebar items, primary buttons).
- Fine-tune transitions for theme-related properties (colors, shadows) to feel smooth but snappy.
